# Завдання 1. Структури даних. Однозв’язний список

## Мета роботи
Реалізувати однозв’язний список та основні алгоритмічні операції над ним:
- реверсування списку;
- сортування списку;
- об’єднання двох відсортованих списків.

Реалізація виконана з використанням об’єктно-орієнтованого підходу.

---

## Реалізовані компоненти

### 1. Однозв’язний список
Створено класи:
- `Node` — вузол списку, який містить значення та посилання на наступний вузол;
- `LinkedList` — клас для управління списком.

---

### 2. Реверсування списку
Реалізовано метод `reverse()`, який виконує реверсування однозв’язного списку **in-place** шляхом зміни посилань між вузлами.

**Особливості:**
- не створюється новий список;
- не використовуються допоміжні структури даних;
- змінюються лише посилання `next`.

**Складність:**
- час: `O(n)`
- пам’ять: `O(1)`

---

### 3. Сортування списку
Для сортування використано алгоритм **merge sort**, який є оптимальним для однозв’язних списків.

**Причини вибору алгоритму:**
- не потребує доступу за індексом;
- ефективний для зв’язаних структур даних;
- має стабільну асимптотичну складність.

**Складність:**
- час: `O(n log n)`
- пам’ять: `O(log n)` (глибина рекурсії)

---

### 4. Об’єднання двох відсортованих списків
Реалізовано метод `merge_with(other)`, який об’єднує два відсортовані однозв’язні списки в один відсортований список.

**Складність:**
- час: `O(n + m)`
- пам’ять: `O(1)`

---

## Висновок
У межах завдання реалізовано повнофункціональний однозв’язний список та основні алгоритмічні операції над ним.
